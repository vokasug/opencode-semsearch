# Examples

## В OpenCode (LLM-вызовы)

### Простой поиск

> «Найди сессию где я что-то возился с three.js»

Модель вызовет:
```json
{
  "query": "three.js solar system"
}
```

Ответ LLM (после саммари результатов):
> Нашёл: 4 сессии про three.js solar system...

### Только вопросы пользователя за последнюю неделю

> «Что я спрашивал про провайдера alibaba на этой неделе?»

```json
{
  "query": "провайдер alibaba",
  "role": "user",
  "since_days": 7
}
```

### Поиск ассистентских ответов

> «Где я раньше объяснял как починить SSH?»

```json
{
  "query": "SSH host identification warning",
  "role": "assistant"
}
```

## Из терминала

```bash
# Прямой запрос
oc-semsearch query "three.js solar system"

# С параметрами
oc-semsearch query "alibaba провайдер" --role user --since-days 7 --top 5

# Только активные сессии (по умолчанию)
oc-semsearch query "что-то"

# Включить архивные
oc-semsearch query "что-то" --no-active-only

# Статистика индекса
oc-semsearch stats

# Self-check
oc-semsearch doctor

# Перестроить индекс
oc-semsearch build

# Переиндексировать конкретные сессии
oc-semsearch reindex ses_0c7a36d7fffe ses_0c70a4866ffe
```

## Типичные фразы для LLM

| Что говоришь | Что модель поймёт |
|---|---|
| «Найди сессию про X» | семантический поиск |
| «Где я спрашивал про Y» | поиск user-сообщений |
| «Покажи прошлый раз когда я возился с Z» | поиск по теме |
| «В каких сессиях обсуждали X» | поиск по теме |
| «Найди сообщение где я писал про Y» | поиск user-сообщений |
| «Какие сессии про X» | семантический поиск |
| «Когда я последний раз делал X» | поиск + сортировка по дате |

## Anti-patterns (НЕ ищи через семантику)

- «Какая последняя сессия» — обычный `opencode session list` (SQL по `time_created`)
- «Сколько сессий про X» — `COUNT(*)` по индексу
- «Покажи код из сессии X» — прочитать напрямую через `read` по session_diff/

Семантический поиск — для нахождения **по смыслу**, не для структурных запросов.
